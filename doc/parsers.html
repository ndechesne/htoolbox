Parsers

Introduction

Parsers are there to save time when dealing with files under version control, typically source code and documents from industry projects.

Version control systems are used to keep track of changes on files deemed important to the project. It is this notion that parsers rely on. Indeed, if files deemed important are under version control, then those that are not controlled are therefore not considered important.

On the other hand, files that are under version control and have not been modified need to necessarily be backed up, as they are already backed up on the version control server, at least for centralised version control systems.

To try and suit all cases, the following parser types have been devised:

Parser type Description
controlled Backs up all files under version control
modified Backs up files under version control that are locally modified
others Backs up all files that are not under version control
local Backs up all files that are not under version control, or are locally modified (= modified + others)

The parsers available are as follows:

Parser name Description Comment
cvs Parser for the CVS version control tool As the parser cannot check whether the files are modified ot not, modified is the same as controlled, and local is the same as having no parser at all
